4.

We'll start with the main part of all this: the compile function. What this function does is whenever the compile button is pressed, it will grab the code that is currently in the editor, and send it to a php script on the server. The php script will run lit on the code, and write the HTML to a file called generated.html. It will return the generated code to the calling function.

The reason we write the HTML out to a file and don't also return that, is because we need to display the html in an iframe, and that works best if it's in its own file.

We'll create the php script right here.

{lit.php 4} 
<?php
// Get the code passed from javascript
$lit_file = $_POST["code"];
// Run lit to get the html

$myfile = fopen("file.lit", "w") or die("Unable to open file!");
fwrite($myfile, $lit_file);
fclose($myfile);

$lit_file = str_replace("'", '\'"\'"\'', $lit_file);

$html = shell_exec('/home/zbyedidia/Literate/bin/lit -w <<< \''. $lit_file . '\'');
$errors = shell_exec('/home/zbyedidia/Literate/bin/lit file.lit');

// If there was an error, set the HTML equal to just the error
if ($errors) {
    $html = str_replace("\n", "<br>", $errors);

    // Open generated.html and write the html to it
    $myfile = fopen("file.html", "w") or die("Unable to open file!");
    fwrite($myfile, str_replace("\n", "<br>", $errors));
    fclose($myfile);
}

// Run lit again to generate the code and return that to javascript (with echo)
echo shell_exec('/home/zbyedidia/Literate/bin/lit -t <<< \''. $lit_file . '\'');
?>

Finally, we make the main function. We wrap all of this in the document ready function.

In this function, we do some set up. We set the editor mode to literate. We resize ace the be the correct width and we set a callback so that whenever the window changes size the resizeWindow function is called again. We also do some set up on the filetree so that whenever a file is clicked, an ajax request for its contents is made, and the resulting content gets put in the ace editor.

{The main function 13} 
$(document).ready(function() {
    var editor = ace.edit("editor");
    // Set the ace mode to literate (written by me)
    editor.getSession().setMode("ace/mode/literate");

    // Create a callback so whenever the window is resized, ace will be resized

15.

Now we have to create the two functions which toggle vim and emacs keybindings. The way we detect if either vim or emacs is enabled in order to toggle is we look to see first if editor.getKeyboardHandler().$id is defined, and if it is we make sure vim or emacs is part of the $id (the full id would be ace/keyboard/vim).

{ToggleVim function 15} 
function toggleVim() {
    var editor = ace.edit("editor");
    var id = editor.getKeyboardHandler().$id;
    console.log(editor.getKeyboardHandler())
    if (id && editor.getKeyboardHandler().$id.indexOf("vim") > -1) {
        editor.setKeyboardHandler("ace/keyboard/keybinding");
        $('#vim-btn').text("Enable Vim key bindings");
    } else {
        editor.setKeyboardHandler("ace/keyboard/vim");
        $('#vim-btn').text("Disable Vim key bindings");
    }
}

This code is used in section 3

{ToggleEmacs function 15} 
function toggleEmacs() {
    var editor = ace.edit("editor");
    var id = editor.getKeyboardHandler().$id;
    if (id && editor.getKeyboardHandler().$id.indexOf("emacs") > -1) {
        editor.setKeyboardHandler("ace/keyboard/keybinding");
        $('#emacs-btn').text("Enable Emacs key bindings");
    } else {
        editor.setKeyboardHandler("ace/keyboard/emacs");
        $('#emacs-btn').text("Disable Emacs key bindings");
    }
}
